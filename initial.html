Process

From Question to
Reproducible Research

Eight intelligent agents work in sequence to classify your intent, retrieve authoritative data, FAIRify it against your metadata schemas, compute statistics, and deliver a citable notebook.

1

Intent Classification

Your natural-language query is classified into one of six intent types — factual lookup, comparison, trend analysis, dataset discovery, methodology question, or data linking — and routed to the appropriate processing tier (1, 2, or 3).

FACTUAL_LOOKUP COMPARISON TREND_ANALYSIS DATASET_DISCOVERY METHODOLOGY_QUESTION DATA_LINKING
2

Entity Extraction

Places are resolved to canonical identifiers, variables are mapped to authoritative codes from the active connector pack (e.g., BLS series, Census variables, FRED codes), and time periods are normalized. Geographic resolution scales from country down to state/province, county, and city — wherever the connectors reach.

geoId/48 → Texas NUTS:FR10 → Île-de-France UnemploymentRate 2019–2024
3

Multi-Source Data Retrieval

A prioritized retrieval strategy first queries Data Commons, then falls back to vector similarity search, then directly calls the configured connector APIs — BLS, Census, BEA, FRED, or any custom source plugged into the connector library. Results are deduplicated and cached with per-source TTLs.

Data Commons v2 Qdrant / Pinecone Redis TTL Cache
4

FAIRification & Metadata Enrichment

Retrieved data is enriched in-process against FAIR principles — Findable, Accessible, Interoperable, Reusable. Customizable metadata schemas (DCAT-AP, Schema.org/Dataset, DataCite, or your organization's own) describe provenance, licensing, controlled vocabularies, and quality flags before any downstream computation.

DCAT-AP Schema.org/Dataset DataCite Custom Schemas
5

Statistical Computation

The Stats Computer runs the appropriate calculation for each intent type: direct lookups, time-series trend analysis, cross-sectional comparisons, or statistical inference — each with its own reliability score.

direct_lookup: 1.0 trend_analysis: 0.85 statistical_inference: 0.70
6

Visualization Generation

Vega-Lite chart specifications are generated automatically — time-series line charts, bar comparisons, geographic choropleth maps — and embedded directly in the output notebook.

Vega-Lite Time Series Choropleth
7

Citation Building

Every data point is traced back to its authoritative source. Citations are formatted in both government and academic styles with publication dates, dataset names, API endpoints, and access timestamps.

Government Format Academic Format API Attribution
8

Reproducible Notebook

Every query generates a self-contained, Colab-ready Jupyter notebook. The full execution trace — every API call, every computation — is recorded so your analysis can be reproduced, modified, and peer-reviewed.

.ipynb Colab-Ready execution_trace
Capabilities

Built for Rigorous
Data Research

Every feature is designed around a core principle: authoritative data, traceable methods, and reproducible results.

🧠

Multi-Agent AI Orchestration

LangGraph-powered supervisor coordinates eight specialized agents — each expert in a different phase of the data pipeline. The supervisor delegates entity extraction, function calling, and reasoning to your configured LLM.

🔐

Bring Your Own LLM

Data Concierge is LLM-agnostic by design. Run with frontier models from your preferred provider, open-weights models, or fully local LLMs deployed on-premises — keeping sensitive data and queries inside your security perimeter.

🧬

Auto-Derived Ontologies

Domain ontologies are derived automatically by combining statistical analysis — distributions, cardinality, type inference, correlations — with FAIRified metadata, in the context of the LLM's general world knowledge. Outputs map cleanly to well-known schemas and vocabularies (Schema.org, SKOS, DCAT, Dublin Core, PROV-O) and can be easily customized and tailored to your domain.

📓

Reproducible Jupyter Notebooks

Every answer ships as a Colab-ready .ipynb file. The full execution trace — API calls, computations, chart specs — is baked in so anyone can reproduce, verify, or extend your analysis.

🔍

Semantic Dataset Discovery

Can't find the right dataset? The CKAN agent combines LLM function calling with Pinecone semantic vector search to find datasets across thousands of open-data catalogs — federal, state, municipal, and international.

📊

Automatic Visualizations

Vega-Lite chart specs are generated automatically for every result — time-series trends, geographic comparisons, distribution plots — and rendered directly in the output notebook.

📎

Government-Grade Citations

Every data point is traceable. Citations include the source agency, dataset name, API endpoint, access date, and series identifier — formatted in both government and academic styles.

⚡

Intelligent Caching

Redis-backed cache with per-source TTLs (BLS: 30 days, Census: 90 days, NCSES: 365 days). Version-aware keys ensure cached data is never stale after API updates.

🎯

Confidence Scoring

A five-factor weighted confidence model grades every answer: query interpretation, source authority, retrieval match quality, data recency, and computation reliability. Transparent, auditable scores.

🔄

Tiered Processing

Simple factual lookups are fully automated (Tier 1). Comparisons and trend analyses run a multi-step pipeline (Tier 2). Complex or low-confidence queries escalate to human review (Tier 3).

🗄️

Session Memory

Conversational sessions keep recent queries in context, enabling follow-up questions and iterative refinement without repeating yourself. Context depth and expiry policy are fully configurable per deployment.

👥

Expert-in-the-Loop Curation

Generated notebooks can be submitted to a GitHub-based review workflow where domain experts vet methodology and verify citations. Approved notebooks join an ever-expanding library of vetted, reproducible, hallucination-free deterministic answers — searchable, citable, and reusable across your organization.
